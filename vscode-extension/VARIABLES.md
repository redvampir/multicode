# Система переменных в MultiCode

> **Blueprint-style Variables** — как в Unreal Engine

## 📋 Обзор

Система переменных в MultiCode реализована по образцу Unreal Engine Blueprints:
- Переменные определяются в графе
- Имеют тип данных (int32, float, string, bool, etc.)
- Могут иметь значение по умолчанию
- Локализованы (русские/английские названия)

## 🎨 Интерфейс

### Панель переменных (Variables Panel)

- **Открыть/закрыть**: кнопка `V` на тулбаре
- **Создать переменную**: кнопка `+ Переменная`
- **Редактировать**: кнопка ✏️ рядом с переменной
- **Удалить**: кнопка 🗑️ (с подтверждением)

### Создание переменной

1. Откройте панель переменных (`V`)
2. Нажмите `+ Переменная`
3. Введите:
   - **Название (EN)** — для кода (латиницей)
   - **Название (RU)** — для отображения
   - **Тип данных** — int32, float, string, bool, etc.
   - **Значение по умолчанию** (опционально)

### Использование в графе

#### Get Variable (Получить переменную)
- Узел для чтения значения переменной
- Автоматически создаётся drag-and-drop из панели
- **Выход**: значение переменной

#### Set Variable (Установить переменную)
- Узел для записи значения
- **Вход exec**: поток выполнения
- **Вход value**: новое значение
- **Выход exec**: продолжение выполнения
- **Выход value**: новое значение (для цепочек)

## 🔧 Технические детали

### Структура данных

```typescript
interface BlueprintVariable {
  id: string;              // Уникальный ID
  name: string;            // Название (EN)
  nameRu: string;          // Название (RU)
  dataType: PortDataType;  // Тип данных
  defaultValue?: string | number | boolean;
  description?: string;    // Описание
  isExposed?: boolean;     // Экспортируется ли
  category?: string;       // Категория
}
```

### Типы данных

| Тип | Описание | Цвет | C++ тип |
|-----|----------|------|---------|
| `int32` | Целое число | Cyan | `int32_t` |
| `int64` | Большое целое | Cyan (темнее) | `int64_t` |
| `float` | Дробное число | Зелёный | `float` |
| `double` | Двойная точность | Зелёный (темнее) | `double` |
| `bool` | Логическое | Красный | `bool` |
| `string` | Строка | Розовый | `std::string` |
| `vector` | Вектор | Жёлтый | `Vector3` |
| `object` | Объект | Синий | `Object*` |
| `array` | Массив | Оранжевый | `std::vector<T>` |
| `any` | Любой | Серый | `std::any` |

## 📝 Примеры использования

### Пример 1: Счётчик

```
Variables:
  - counter: int32 = 0

Graph:
  Start
    ↓
  Get counter → Add (B=1) → Set counter
    ↓
  Print (counter)
    ↓
  End
```

### Пример 2: Приветствие

```
Variables:
  - userName: string = "User"
  - greeting: string = ""

Graph:
  Start
    ↓
  Input ("Имя:") → Set userName
    ↓
  Get userName → Concat ("Привет, ", userName) → Set greeting
    ↓
  Get greeting → Print
    ↓
  End
```

### Пример 3: Условная логика

```
Variables:
  - health: float = 100.0
  - isAlive: bool = true

Graph:
  Start
    ↓
  Get health → Less (B=0) → Branch
    True → Set isAlive (false) → Print ("Game Over")
    False → Print ("Continue")
    ↓
  End
```

## 🎯 Best Practices

1. **Именование**:
   - EN: camelCase (`playerHealth`, `scoreCounter`)
   - RU: Русские названия (`Здоровье игрока`, `Счётчик очков`)

2. **Типы**:
   - Используйте конкретные типы (`int32` вместо `any`)
   - `float` для координат и процентов
   - `bool` для флагов состояния

3. **Значения по умолчанию**:
   - Всегда задавайте для числовых типов (избегайте undefined)
   - Для строк: `""` или осмысленное значение

4. **Организация**:
   - Группируйте по категориям
   - Удаляйте неиспользуемые переменные

## 🔄 Синхронизация с узлами

При изменении переменной (имя, тип, значение):
- ✅ Все узлы Get/Set автоматически обновляются
- ✅ Названия портов синхронизируются
- ✅ Типы данных проверяются
