# Миграция схемы графа для UE-ветки (schemaVersion=3)

## Что меняется

- В сериализованном документе графа вводится явное поле `schemaVersion`.
- Текущая целевая версия схемы: `3`.
- Для обратной совместимости поддерживается чтение legacy-документов без `schemaVersion` (`version=1/2`).
- UE-поля в классах (`extensions.ue.classMacro`, `generatedBodyMacro`, `methodMacro`) считаются **опциональными** в исходных данных и при миграции заполняются безопасными значениями по умолчанию.

## Breaking / Non-breaking матрица

| Сценарий | Статус | Комментарий |
| --- | --- | --- |
| Открытие старых `.multicode/.json` без `schemaVersion` | Non-breaking | Поддерживается мигратором. Поля `integrationBindings`, `symbolLocalization`, `graphVersion`, UE-metadata нормализуются автоматически. |
| Экспорт в modern-режиме (`schemaVersion=3`) в старые внешние пайплайны | Потенциально breaking | Старый consumer может ожидать только `version` и не знать `schemaVersion`. |
| Экспорт в legacy-режиме | Non-breaking | Формат соответствует предыдущему контракту (`version=2`, без `schemaVersion`). |
| Повторная загрузка новых UE-графов | Non-breaking | UE-метаданные сохраняются без деградации. |

## План миграции

1. **Чтение**: принимать оба формата (modern + legacy).
2. **Нормализация**: при десериализации приводить документ к `schemaVersion=3` в памяти.
3. **Экспорт**:
   - `legacy` — для старых пайплайнов;
   - `modern` — для нового контракта и полного набора метаданных.
4. **Релизный контроль**: перед pre-release UE-ветки пройти чек-лист `Документы/Инфраструктура/UE_PRE_RELEASE_CHECKLIST.md`.

## Как откатиться

Если после включения UE-ветки внешний пайплайн не принимает новый экспорт:

1. В настройках VS Code выставить `multicode.graphExport.compatibilityMode = legacy`.
2. Пересохранить граф и повторить экспорт.
3. Проверить, что в документе отсутствует `schemaVersion` и присутствует `version=2`.

## Как проверить

1. Открыть legacy-граф без UE-полей — загрузка должна быть успешной.
2. Экспортировать тот же граф в режиме `legacy` — структура должна совпасть с прежним контрактом.
3. Загрузить и повторно сохранить modern UE-граф — `classes[].extensions.ue` должен остаться неизменным.
