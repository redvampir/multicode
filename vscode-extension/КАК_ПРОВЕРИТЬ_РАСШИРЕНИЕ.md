# Диагностика расширения MultiCode

> Документ объединяет актуальные шаги проверки. Историческая версия перенесена в `vscode-extension/Архив/test-extension.md`.

## ✅ Текущий статус

Базовый сценарий проверки актуален для релиза `0.4.1` и новых версий:
- ✅ Проверка активации extension
- ✅ Проверка регистрации команд `multicode.*`
- ✅ Проверка работы webview и логов

---

## 🚨 Важно после обновления VSIX

Если вы переустановили расширение, обязательно перезагрузите окно VS Code, иначе будет работать старая версия кода.

---

## Шаг 1: Перезагрузка VS Code

1. Нажми `Ctrl+Shift+P`
2. Введи: `Developer: Reload Window`
3. Нажми Enter

> VS Code активирует расширения при старте окна. После установки новой сборки нужна перезагрузка.

---

## Шаг 2: Открой Developer Tools

1. Нажми `Ctrl+Shift+P`
2. Введи: `Developer: Toggle Developer Tools`
3. Переключись на вкладку **Console** (не Elements, не Network!)

---

## Шаг 3: Проверь Логи Активации

В консоли должны появиться сообщения:

```
[MultiCode] ========================================
[MultiCode] Extension ACTIVATION started
[MultiCode] extensionPath: C:\Users\...\multicode-team.multicode-visual-programming-<version>
[MultiCode] ========================================
[MultiCode] Registering command: multicode.openEditor
[MultiCode] Registering command: multicode.newGraph
[MultiCode] Registering command: multicode.saveGraph
[MultiCode] Registering command: multicode.loadGraph
[MultiCode] Registering command: multicode.generateCode
[MultiCode] Registering command: multicode.generateCodeBinding
[MultiCode] Registering command: multicode.translateGraph
[MultiCode] Registering command: multicode.compileAndRun
[MultiCode] All commands registered successfully!
[MultiCode] Registered commands: ['multicode.openEditor', ...]
```

### Что Искать:

| Что видишь | Что это значит | Что делать |
|------------|----------------|------------|
| ✅ Все логи `[MultiCode]` присутствуют | Расширение активируется корректно | Переходи к Шагу 4 |
| ❌ Нет логов `[MultiCode]` вообще | Расширение НЕ активируется | Смотри раздел "Если Расширение Не Активируется" |
| ⚠️ Есть `ACTIVATION started`, но нет `All commands registered` | Ошибка при регистрации команд | Смотри полный лог ошибки в консоли |

---

## Шаг 4: Попробуй Команду

1. Нажми `Ctrl+Shift+P`
2. Введи: `MultiCode: Open Visual Editor`
3. Нажми Enter

### Результат:

- **Успех:** Открылся визуальный редактор
- **Ошибка:** Снова "command not found"

Если ошибка - переходи к Шагу 5.

---

## Шаг 5: Проверь Output Panel

1. Открой: `View → Output` (или `Ctrl+Shift+U`)
2. В выпадающем списке выбери: **MultiCode**
3. Проверь наличие:

```
[MultiCode] Extension activated!
[MultiCode] Timestamp: 2025-12-14T...
[MultiCode] Extension activation complete!
```

## 🔍 Диагностика Проблем

### Если Расширение Не Активируется (нет логов)

1. **Проверь список активных расширений:**
   - `Ctrl+Shift+P` → `Developer: Show Running Extensions`
   - Найди `multicode-team.multicode-visual-programming`
   - Проверь статус активации

2. **Проверь ошибки активации:**
   - В Developer Tools перейди в **Console**
   - Посмотри на красные сообщения об ошибках
   - Скопируй текст ошибки

3. **Ручная проверка установки:**
   ```powershell
   code --list-extensions | Select-String "multicode"
   ```

### Если Команды Не Регистрируются (есть логи активации, но нет "All commands registered")

1. **Проверь полный лог в Console:**
   - Найди где прерывается последовательность логов
   - Посмотри на ошибку JavaScript

2. **Проверь доступность команд вручную:**
   - Открой Developer Tools → Console
   - Выполни:
     ```javascript
     vscode.commands.getCommands(true).then(cmds => {
       console.log('MultiCode commands:', 
         cmds.filter(c => c.startsWith('multicode.'))
       );
     });
     ```
   - Если массив пуст → команды не зарегистрированы
   - Если команды есть → проблема в манифесте или title

### Если команды зарегистрированы, но VS Code их не показывает

Обычно это проблема сборки `dist/extension.js` или кэша расширения.

1. **Проверь dist/extension.js:**
   ```powershell
   Get-Content dist\extension.js | Select-Object -Last 10
   ```
   
2. **Посмотри на структуру экспорта:**
   - Должно быть: `module.exports = ...` или `exports.activate = ...`
   - Если видишь `module.exports=t` (одна буква) → webpack минификация сломала экспорты

3. **Отключи минификацию для теста:**
   - Открой `webpack.config.js`
   - Найди секцию `optimization`
   - Добавь:
     ```javascript
     optimization: {
       minimize: false
     }
     ```
   - Пересобери: `npm run package`

## 🛠️ Быстрые Команды для Отладки

### Пересборка без минификации (диагностика):
```powershell
# В webpack.config.js установи optimization.minimize = false
npm run package
```

### Переустановка:
```powershell
$vsix = Get-ChildItem "multicode-visual-programming-*.vsix" | Sort-Object LastWriteTime -Descending | Select-Object -First 1
code --uninstall-extension multicode-team.multicode-visual-programming
code --install-extension $vsix.FullName --force
```

### Проверка логов в терминале:
```powershell
.\verify-extension.ps1
```

После перезагрузки VS Code и открытия Developer Tools ты должен увидеть:

✅ Логи активации в Console
✅ Основные команды зарегистрированы
✅ Сообщения в Output Panel → MultiCode
✅ Команда `MultiCode: Open Visual Editor` работает

Если этого не происходит - логи покажут **где именно** ломается процесс.
