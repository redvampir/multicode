# Функционал - Компиляция и Запуск C++ Кода

**Версия:** 0.4.1+  
**Статус:** ✅ Реализовано  
**Дата:** 23 февраля 2026

## Описание

Добавлен механизм компиляции и запуска C++ кода, сгенерированного плагином MultiCode, с максимально бесшовным UX:
- расширение пытается использовать системный компилятор, если он реально умеет **C++23**
- если подходящего компилятора нет — один раз предлагает скачать и установить **managed toolchain** (без прав администратора)
- после установки компиляция/запуск работают без участия пользователя

## Основные возможности

### 1. **Стандарт запуска: всегда C++23 (strict)**

Кнопка **▶️ Запустить** всегда компилирует в режиме **C++23 strict**:
- стандарт не понижается автоматически (нет fallback на C++20/17/14)
- если системный компилятор не поддерживает C++23 — используется managed toolchain

Значение настройки `multicode.cpp.standard` сохранено для совместимости, но не влияет на Run.

### 2. **Компиляция и Запуск**

Новая кнопка **▶️ Запустить** в toolbar:
- Генерирует код из визуального графа
- Компилирует в исполняемый файл
- Запускает программу и выводит результат
- Автоматически удаляет временные файлы

**Где находится:** Группа "Код" в toolbar → кнопка после "Генерировать"

## Архитектура

### Модули

#### `src/compilation/CppCompiler.ts`
Основной модуль компиляции:
- **findCompiler()** — поиск компилятора на системе (MSVC, GCC, Clang)
- **compileCpp()** — компиляция C++ файла (поддерживает strict C++23, env/PATH и extraArgs)
- **executeProgram()** — запуск исполняемого файла
- **cleanupTempFiles()** — удаление временных файлов

#### `src/compilation/ToolchainManager.ts`
Менеджер тулчейнов для Run:
- probe-компиляция для проверки реальной поддержки C++23
- выбор: `system` (если подходит) или `managed` (скачать и использовать)
- верификация скачанного архива по sha256 (pinned источник)

#### `src/panel/GraphPanel.ts`
Интеграция в VS Code расширение:
- `handleCompileAndRun()` — новый публичный метод
- Вызывает `ensureCppToolchain()` перед компиляцией
- Запускает **строго C++23**
- Вывод логов в Output Channel

#### `src/webview/main.tsx`
UI компоненты:
- Кнопка "Запустить"
- Интеграция с командой `requestCompileAndRun`

#### `src/extension.ts`
Команда расширения:
- `multicode.compileAndRun` — новая команда

### Поток данных

```
Visual Graph (Blueprint)
    ↓
    Generate C++ Code 
    ↓
    Save to Temp File
    ↓
    Ensure Toolchain (System C++23 → иначе Managed)
    ↓
    Compile (strict C++23)
    ↓
    Execute Binary
    ↓
    Capture Output
    ↓
    Display in Output Channel
    ↓
    Cleanup Temp Files
```

## Поддерживаемые компиляторы

| Компилятор | Поддержка | Стандарты | Команда |
|-----------|---------|----------|---------|
| MSVC (Visual Studio) | ✅ | C++17, C++20, C++latest | `cl` |
| GCC | ✅ | C++17, C++20, C++23 | `g++` |
| Clang | ✅ | C++17, C++20, C++23 | `clang++` |

**Автоопределение:** пробуем компиляторы в порядке **MSVC → Clang → GCC**, выбирая тот, который реально может собрать текущий код в режиме **C++23 strict**.

Если системный компилятор не найден или не поддерживает C++23 — расширение предлагает установить managed toolchain (один раз, далее без вопросов).

### Managed toolchains (без admin/root)
Текущие источники (pinned):
- Windows: `llvm-mingw` (clang++ + mingw-w64 runtime)
- Linux x64: Bootlin Buildroot toolchain (musl)
- Linux arm64: LLVM (clang) + Bootlin sysroot (musl)
- macOS: требуется Xcode Command Line Tools; на arm64 доступен managed LLVM tarball

## Примеры использования

### Простой граф: "Hello World"

1. Создайте граф с узлами:
   - Start
   - Print ("Hello, World!")
   - End

2. Нажмите **▶️ Запустить**

3. В Output Channel появится:
   ```

   [Compile & Run] ✓ Код успешно сгенерирован
   ═══════════════════════════════════════════════════════════════
   [Compile & Run] Стандарт C++: cpp23 (strict)
   [Compile & Run] Компиляция...
   [Compile & Run] ✓ Компиляция успешна (245мс)
   [Compile & Run] Запуск программы...
   ────────────────────────────────────────────────────────────────
   Hello, World!
   ────────────────────────────────────────────────────────────────
   [Compile & Run] ✓ Программа завершена (exit code: 0, 15мс)
   ```

### Использование C++23-фич

Стандарт уже фиксирован на **C++23**. Если компиляция падает из-за того, что системный компилятор слишком старый:
- согласитесь на установку managed toolchain (предложение появится автоматически при Run)
- либо обновите системный компилятор

## Обработка ошибок

### Ошибки генерации
```
[Compile & Run] ✗ Ошибка генерации кода:
  - Узел Start не найден
```
→ Проверьте граф, убедитесь в наличии узла Start

### Ошибки компиляции
```
[Compile & Run] ✗ Ошибка компиляции:
error: 'std::spaceship' not defined for this version
```
→ Обновите компилятор или используйте managed toolchain (Run предложит установку автоматически)

### Компилятор не найден
```
[Compile & Run] ✗ Ошибка: Компилятор C++ не найден. 
Установите MSVC, GCC или Clang.
```
→ При Run расширение предложит скачать managed toolchain. Если вы отказались — проверьте настройки `multicode.cpp.toolchain.*`.

## Процесс компиляции

### Флаги компиляции

**GCC/Clang:**
```bash
g++ -std=c++20 -O2 -o output source.cpp
```

**MSVC:**
```bash
cl /std:c++20 /O2 /Feoutput.exe source.cpp
```

### Оптимизация

По умолчанию используется **-O2 (Release)**, что обеспечивает:
- Оптимальное соотношение размера код / производительность
- Быструю компиляцию
- Поддержку полезных предупреждений

## Безопасность

### Временные файлы
- Сохраняются в системный временный каталог (`/tmp` на Unix, `C:\Temp` на Windows)
- Название: `multicode_temp_<timestamp>.cpp`
- Автоматически удаляются после выполнения
- *Не сохраняются на диск при нормальном завершении*

### Запуск кода
- Исполняемый файл создается в том же каталоге, что и исходник
- Выполняется в песочнице временной папки
- Нет доступа к исходному коду графа

## Производительность

| Операция | Средний Время |
|---------|--------------|
| Генерация кода | 5-20 мс |
| Компиляция простой программы | 200-400 мс |
| Запуск программы | 10-50 мс |
| **Итого (простая программа)** | **~300-500 мс** |

## Ограничения

1. **Стандартная библиотека только:** Нет поддержки внешних библиотек (boost, Qt и т.д.)
2. **Только консольный вывод:** GUI программы не поддерживаются
3. **Таймаут 10 сек:** Программы, работающие дольше 10 секунд, будут прерваны
4. **Только C++:** Нет поддержки Rust/Assembly (пока)
