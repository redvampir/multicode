## Документация и контракты
- Держать активные документы синхронно с `vscode-extension/package.json`, `ROADMAP.md` и фактической структурой `vscode-extension/src/`.
- Явно развести два формата: C++ `GraphSerializer` и `.multicode`-сериализацию webview, чтобы команда не путала core snapshot с editor persistence.
- Зафиксировать, какие документы нормативные, а какие исторические; активные файлы не должны описывать уже реализованные модули как “MVP” или “план”.

## Core / Graph API
- Свести к минимуму неоднозначные публичные контракты вокруг ID, ошибок и владения указателями.
- Продолжить движение к более явным `Result<T>`/кодам ошибок на границах API, особенно там, где extension-side должен показывать понятную причину сбоя.
- Отдельно оценить потокобезопасность графа и необходимость контроллера/lock-обёртки до выхода на более тяжёлые сценарии и большие графы.

## Сериализация и миграции
- Убрать рассинхрон между документацией, публичными константами и фактическим runtime-форматом `GraphSerializer`.
- Формализовать стратегию миграций для `.multicode` и C++ graph JSON: какие версии поддерживаются, где делается upgrade, какие кейсы считаются legacy.
- Добавить утилиту или сценарий проверки “загрузить старый граф -> нормализовать -> сохранить -> сравнить ожидаемый результат”.

## Code Generation / Targets
- Дожать паритет между `cpp` и `ue` там, где общий graph model уже совпадает, а расхождения должны жить только в target strategy.
- Продолжить покрывать golden-тестами сложные кейсы: пользовательские функции, class sidecars, pointer/reference, dependency-driven nodes.
- Для `rust` и `asm` сохранить честный статус “не поддерживается”, пока не появится реальный генератор и тестовый контур.
- Стабилизировать контракты предупреждений (`warnings`) и source mapping, чтобы preview, запись в файл и compile/run сообщали одну и ту же картину.

## VS Code Extension / UX
- Полировать `Dependency View v2`, `Class System v2` и class sidecar flows без разрастания UI-хаоса.
- Следить за паритетом между `BlueprintEditor` и `GraphEditor` в общих сценариях: загрузка, валидация, переключение, сохранение, preview.
- Упростить большие панели и длинные списки через виртуализацию, фильтры и более предсказуемые пустые/ошибочные состояния.
- Отдельно контролировать UX привязки графа к исходникам и class files: пользователь должен понимать, где источник истины и что именно было восстановлено/перечитано.

## CI / Tooling
- Добавить лёгкую проверку документации на явные рассинхроны: версия, битые ссылки, упоминания архивных состояний как актуальных.
- Поддерживать быстрый smoke path и полный path отдельно, чтобы CI не становился дорогим при каждом изменении webview/docs.
- Улучшить кеширование npm/vcpkg/CMake и сократить время feedback loop для PR.

## Testing
- Расширять golden/regression tests для `vscode-extension/src/codegen/` и `tests/generators/`.
- Добавить сценарии на сериализацию и обратную совместимость для `.multicode` и C++ graph JSON.
- Усилить regression-покрытие для UX-фич, которые ломаются незаметно: undo/redo, copy/paste, editor switching, class sidecars, dependency drag-and-drop.
- Держать минимальный набор smoke-проверок документации и команд из README, чтобы “быстрый старт” не расходился с репозиторием.
