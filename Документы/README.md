# Документация MultiCode (актуальная версия)

Эта папка больше не склад фантазий. Здесь собраны только те материалы, которые описывают фактическое состояние проекта или конкретные задачи, которые предстоит выполнить.

## Быстрый индекс
| Файл | Назначение |
| --- | --- |
| `Архитектура/README.md` | Слои системы: ядро на C++, планы на сериализацию, биндинги и UI. |
| `Архитектура/PackageSystem.md` | Что такое пакет узлов, какие артефакты нужны, что уже сделано (список пуст) и какой минимум, чтобы собрать POC. |
| `Архитектура/AdvancedFeatures.md` | Фичи «в перспективе» (AI naming, live preview и т.д.) с пометкой, реализованы ли они. |
| `Архитектура/Multithreading.md` | Модель потоков: что уже потокобезопасно, где нужны локи, какие задачи впереди. |
| `Архитектура/VisualEditor.md` | Как должен выглядеть webview-редактор VS Code и что пока отсутствует. |
| `Архитектура/CodeBinding.md` | Будущая связка граф ↔ исходники (stub-состояние). |
| `Инфраструктура/README.md` | Регламент Renovate и политика обновления зависимостей. |
| `ProjectStatus.md` | Таблица «что работает / что нет» + риски на ближайший спринт. |
| `ProjectReview.md` | Банковская выписка по недавним решениям: что оставляем, что выкинули, почему. |
| `Roadmap.md` | Укороченный бэклог без воды: версии 0.1/0.2/0.3 и критерии готовности. |
| `ANNOUNCEMENT.md` | Сообщение для команды: что можно трогать прямо сейчас. |
| `BUILD_STATUS.md` | Чеклист инфраструктуры (CI, форматирование, тесты). |
| `DEPENDENCIES.md` | Минимальный набор внешних библиотек и как их ставить. |
| `GrowthAreas.md` | Список точек роста по подсистемам: что править в первую очередь. |
| `CriticalThinking.md` | Чек-лист 'зачем/для кого/какой эффект' для любого текста и фичи. |

## Принципы ведения документов
1. **Только проверенные факты**. Если функция не реализована, так и пишем.
2. **Максимум конкретики**: ссылки на файлы, точные команды, статусы тестов.
3. **Обновляем по результату работы**, а не «на будущее».
