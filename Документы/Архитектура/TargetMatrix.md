# Матрица возможностей target-ов кодогенерации

## Состояние на 2026-02

| Target | Статус генератора | Основной рантайм | Поддержка UE-макросов | include-правила | Ограничения |
|---|---|---|---|---|---|
| `cpp` | ✅ production | стандартный C++ (C++23) | ❌ | стандартные `<...>` include | без дополнительных ограничений UE |
| `ue` | ✅ production | Unreal Engine C++ | ✅ `UCLASS`, `UFUNCTION`, `GENERATED_BODY` | `CoreMinimal.h`, `UObject/NoExportTypes.h`, `<Graph>Generated.generated.h` | fail-fast на UE-несовместимых узлах и именах |
| `rust` | ⚠️ заглушка (без полноценной генерации) | Rust | ❌ | N/A | генератор недоступен |
| `asm` | ⚠️ заглушка (без полноценной генерации) | ASM | ❌ | N/A | генератор недоступен |

## Что означает fail-fast для `ue`

До рендера кода выполняется отдельная валидация UE-стратегии:

- запрещённые конструкции (на текущем этапе: `Parallel`);
- невалидные идентификаторы для имён узлов/функций/переменных/классов;
- диагностические ошибки возвращаются с кодом `UE_UNSUPPORTED_CONSTRUCT`.

Это позволяет остановить генерацию до сборки C++ и показать понятную причину пользователю.

## Single-source model для `Class/Field/Method`

- Базовая модель классов для кодогенерации едина для target-ов `cpp` и `ue`.
- UE-специфика добавляется через композицию extension-полей (`extensions.ue`) поверх той же модели, без отдельной UE-копии DTO.
- Пайплайн `graph -> class model -> render` используется общий; target-стратегия меняет только валидацию/обёртку и макросы (`UCLASS`, `UFUNCTION`, `GENERATED_BODY`).
- Это устраняет второй контур маппинга и снижает риск рассинхронизации C++/UE генераторов при эволюции `BlueprintClass`.
