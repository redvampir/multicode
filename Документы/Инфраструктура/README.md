## Зачем нам Renovate
- **Единая точка правды**: состояние зависимостей теперь отслеживается ботом, а статус виден в [доске зависимостей](#рабочий-процесс-разработчика).
- **Меньше ручных апдейтов**: безопасные минорные обновления для npm и vcpkg прилетают автоматически, не отвлекая команду от фичей.
- **Контроль рисков**: критические мейджоры всегда проходят ручную ревизию и получают метку `breaking` для прозрачности.

> 💡 *Renovate* — это сервис, который анализирует `package.json`, `vcpkg.json`, GitHub Actions и обновляет версии зависимостей через pull request. Так мы избегаем «скрытой» устаревшей библиотеки в CI или редакторе.

## Стратегия автообновлений
- **npm runtime зависимости** (webview, zustand, cytoscape) автоматически сливаются при минорных/патч релизах после 7 дней выдержки и успешного CI.
- **npm devDependencies** (webpack, eslint, typescript) собираются в один PR по выходным, без авто-merge, чтобы быстро поймать несовместимости линтеров.
- **vcpkg библиотеки**: минорные/патч-версии (Catch2, spdlog, nlohmann-json) автомерджатся после 5 дней наблюдения; мейджоры требуют обязательного ручного ревью.
- **GitHub Actions**: безопасные изменения (patch/minor) попадают напрямую при зелёном CI, чтобы пайплайн оставался актуальным.
- **Форматтеры (Prettier)** и **TypeScript toolchain** обновляются отдельно, без авто-merge, чтобы не сломать наш pipeline статического анализа.
- **Базовая ревизия vcpkg** отслеживается через `builtin-baseline` — Renovate будет предлагать PR с новым коммитом из `microsoft/vcpkg`.

## Политики безопасности
- Автослияния включены только для патч/минор обновлений в «безопасных» категориях. Все мейджоры проходят ручную проверку и автоматически помечаются лейблом `breaking`.
- Каждый PR получает семантический коммит `chore(deps): …`, так что история остаётся читаемой и совместимой с релизным процессом.
- Встроенный `rollbackPrs` мгновенно откатывает PR, если тесты на свежей версии падают — бот не оставит проект в сломанном состоянии.
- При массовых PR действует лимит: не более 2 заявок в час и 10 одновременно, чтобы CI не встал намертво.

## Рабочий процесс разработчика
1. **Следим за доской**: автоматическая issue «📦 Обновления зависимостей (Renovate)» подсвечивает все открытые задачи.
2. **Приоритизация**: метки `npm`, `c++`, `lint`, `dev`, `formatting` быстро подсказывают, где понадобится ручная проверка.
3. **Ревью**: для мейджоров Renovate автоматически назначает записанных в `CODEOWNERS` — не забываем отписываться после проверки.
4. **Горячие фиксы**: если зависимость ломает сборку, добавьте её в `ignoreDeps` временно и создайте задачу в [ProjectStatus](../ProjectStatus.md), чтобы отследить ручное восстановление.

## Диагностика и улучшения
- **CI красный после автообновления?** Проверьте соответствующий PR — бот создаст откат, но стоит задокументировать корень проблемы в [ProjectReview](../ProjectReview.md).
- **Нужно ускорить обновления?** Поправьте расписание или лимиты в `renovate.json`, а изменения опишите в этом документе для синхронизации команды.
- **Добавляется новый пакетный менеджер?** Расширяйте `enabledManagers` и добавляйте отдельное правило, чтобы не смешивать экосистемы.
