# ⚡ Быстрый Тест: Pointer Variable Binding

## 🎯 Что Тестируем
Выпадающий список для привязки pointer/class переменных в узлах SetVariable.

## 📝 Шаги

### 1. Запуск (10 сек)
```
Ctrl+Shift+P → "MultiCode: Open Visual Graph Editor"
```

### 2. Создание Переменных (20 сек)
Панель **Variables** → **"+ Создать переменную"**:
- `myPointer` — тип **pointer**
- `myClass` — тип **class**  
- `counter` — тип **int32**

### 3. Создание Set Узла (10 сек)
Перетащи `myPointer` на граф → выбери **Set**

**Ожидаемый результат:**
```
┌─ Set: myPointer ─────┐
│ ◇ Exec In            │
│ ◇ Exec Out           │
│ → Value ○            │  ← Пунктирный круг (pointer port)
├──────────────────────┤
│ Привязка к:          │
│ ┌──────────────────┐ │
│ │ ▼ Выбор...       │ │  ← DROPDOWN
│ └──────────────────┘ │
└──────────────────────┘
```

### 4. Проверка Dropdown (15 сек)
Кликни на dropdown → должен показать:
```
— Не выбрано —
myPointer (pointer)
myClass (class)
counter (int32)
```

Выбери любую переменную → значение сохраняется ✅

### 5. Локализация (5 сек)
Toolbar → кнопка **RU** → **EN**

Dropdown метка должна измениться:
```
"Привязка к:" → "Bind to:"
"— Не выбрано —" → "— None —"
```

### 6. Динамическое Обновление (10 сек)
1. Создай новую переменную `test`
2. Вернись к dropdown в Set узле
3. Проверь — список обновился? ✅

### 7. Визуализация Портов (5 сек)
Pointer порты должны иметь:
- ✅ Пунктирную границу (dashed)
- ✅ Стрелки → в имени
- ✅ Синий цвет (#2196F3)

## 🐛 Что Проверить Если Не Работает

### Dropdown не появляется?
1. Проверь тип переменной (должен быть pointer или class)
2. Проверь что порт Value НЕ подключён к другому узлу
3. Проверь консоль браузера (F12)

### Список переменных пустой?
1. Убедись что переменные созданы в графе
2. Проверь `graph.variables` в консоли:
   ```js
   // В консоли браузера (F12)
   window.vscodeApi.getState()
   ```

### Значение не сохраняется?
1. Проверь `node.properties.inputValue` в консоли
2. Проверь что `handlePropertyChange` вызывается (добавь `console.log`)
