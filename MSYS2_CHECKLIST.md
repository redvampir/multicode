# ✅ Чек-лист MSYS2 Установки

## 📋 Что нужно сделать

### Фаза 1: Подготовка (5 минут)

- [ ] **Скачайте MSYS2**
  - Перейти на: https://www.msys2.org/
  - Скачать: **msys2-x86_64-*.exe** (последняя версия)
  - Размер: ~150 МБ

- [ ] **Запомнить путь установки**
  - По умолчанию: `C:\msys64`
  - ⚠️ Не меняйте путь!

### Фаза 2: Установка MSYS2 (5-10 минут)

- [ ] **Запустить установщик**
  - Двойной клик на скачанный `.exe`
  - Выбрать: "Install" (установить)
  - Выбрать путь: `C:\msys64` (по умолчанию)
  - Нажать "Next" → "Install"

- [ ] **Дождаться завершения**
  - ⏳ Установка займет 2-5 минут
  - Распаковка файлов...
  - По завершении: ✓

- [ ] **При появлении диалога "Запустить MSYS2"**
  - Нажать **Finish** (завершить)
  - Оставить галочку "Run MSYS2" ✓

### Фаза 3: Первичное обновление (15-30 минут)

- [ ] **В терминале MSYS2 (должно открыться)**
  ```
  Если не открылось, запустите:
  Start → найдите "MSYS2 UCRT64" → клик
  ```

- [ ] **Первый проход обновления**
  ```bash
  pacman -Syuu --noconfirm
  ```
  - ⏳ Это может занять 5-10 минут
  - Устанавливаются обновления базовой системы
  - Дождаться сообщения: `>>> Packages to remove`
  - Выбрать: `n` (нет, не удалять) или просто Enter
  - Завершится с сообщением: `(nothing to do)`

- [ ] **Перезагрузить MSYS2** (если предложено)
  - Часто MSYS2 просит перезагрузиться после первого обновления
  - Выбрать: Y (yes)
  - Терминал закроется

- [ ] **Открыть MSYS2 снова**
  - Start → "MSYS2 UCRT64"
  - Или нажать Up Arrow в PowerShell и выберить повторное открытие

- [ ] **Второй проход обновления**
  ```bash
  pacman -Syuu --noconfirm
  ```
  - Дождаться завершения

### Фаза 4: Установка компиляторов (10-20 минут)

- [ ] **В том же терминале MSYS2 выполнить**
  ```bash
  pacman -Sy --noconfirm base-devel mingw-w64-ucrt-x86_64-gcc mingw-w64-ucrt-x86_64-gdb mingw-w64-ucrt-x86_64-cmake
  ```
  - ⏳ Установка займет 10-15 минут
  - Будут загружены и установлены:
    - GCC компилятор
    - Make, CMake и другие инструменты
    - GDB отладчик
  - Дождаться завершения

- [ ] **Проверить установку в MSYS2 терминале**
  ```bash
  g++ --version
  gcc --version
  cmake --version
  ```
  - Всё должно отобразить версии

### Фаза 5: Добавление в PATH (5 минут)

**Вариант A: Автоматически (Рекомендуется)**

- [ ] **Откройте PowerShell от администратора**
  - Start → найдите "PowerShell"
  - Клик правой кнопкой → "Run as Administrator" (Запуск от администратора)

- [ ] **Выполните в PowerShell**
  ```powershell
  Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process -Force
  & 'F:\MultiCode\MultiCode_VS\scripts\install-msys2.ps1'
  ```

- [ ] **Дождаться завершения**
  - Скрипт сам обновит PATH
  - На экране должно появиться: ✓ PATH обновлена

**Вариант B: Ручно**

- [ ] **Открыть "Переменные среды"**
  - Start → напечатать "Переменные среды"
  - Нажать Enter
  - Откроется окно

- [ ] **Добавить путь MSYS2**
  - Нажать **Переменные среды** (кнопка снизу)
  - В разделе "Переменные пользователя" найти **Path**
  - Если нет → нажать **Создать**
  - Нажать **Изменить** (если существует)
  - Нажать **Создать** (в окне списка)
  - Напечатать: `C:\msys64\ucrt64\bin`
  - **ВАЖНО:** переместить эту строку ВЫШЕ Strawberry (если есть)
  - Нажать OK на всех окнах

### Фаза 6: Проверка (2 минуты)

- [ ] **Перезагрузить Windows** ⚠️ ОБЯЗАТЕЛЬНО
  - Только после этого изменения PATH вступят в силу!

- [ ] **Открыть новый PowerShell/CMD**
  - Это ВАЖНО! Старый командный процесс в памяти имеет старый PATH

- [ ] **Проверить компилятор**
  ```powershell
  where.exe g++
  ```
  - Должно быть: **C:\msys64\ucrt64\bin\g++.exe** ✓
  - Если вместо этого видите `C:\Strawberry\...` → PATH не обновлена правильно

- [ ] **Проверить версии**
  ```powershell
  g++ --version
  ```
  - Должно быть: **g++ 13.x+** (не 5.x из Strawberry!)

### Фаза 7: Готовность MultiCode (2 минуты)

- [ ] **Перезагрузить VS Code**
  - Закрыть полностью
  - Открыть заново

- [ ] **При необходимости очистить CMake cache**
  - Откройте палитру команд: `Ctrl+Shift+P`
  - Введите: `CMake: Delete Cache`

- [ ] **Готово к использованию! 🎉**

## ❌ Если что-то не работает

### Проблема: "where.exe g++" показывает старую версию

**Действие 1: Перезагрузить Windows**
```powershell
# Это самое частое решение!
```

**Действие 2: Обновить PATH в текущем процессе**
```powershell
$env:PATH = [Environment]::GetEnvironmentVariable("PATH", "Machine")
where.exe g++  # Проверить снова
```

**Действие 3: Проверить систему PATH**
```powershell
[Environment]::GetEnvironmentVariable("PATH", "Machine") -split ";" | Select-String -Pattern "msys64|strawberry"
```
- Должно быть: `C:\msys64\ucrt64\bin` ВЫШЕ `C:\Strawberry\...`

### Проблема: "g++.exe: command not found" в MSYS2

**Решение:** Переустановить компилятор
```bash
# В терминале MSYS2:
pacman -S --noconfirm mingw-w64-ucrt-x86_64-gcc
```

### Проблема: CMake не находит компилятор

**Решение:** 
```bash
cd /path/to/project/build
cmake --fresh -DCMAKE_CXX_COMPILER=C:/msys64/ucrt64/bin/g++.exe ..
```

### Проблема: Установка зависла

**Решение:** Нажать `Ctrl+C` и повторить:
```bash
pacman -Syuu --noconfirm
```
